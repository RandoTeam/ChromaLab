Status: `DR_C1_COMPLETE`

## Purpose

DR-C1 prepares the graph/layout/axis research wave. It does not change Android
runtime, graph detection, calibration, trace extraction, peak integration,
`CalculationEngine`, chromatographic math, validators, model routing, or report
rendering.

The goal is to stop comparing algorithms by eye. Before DR-C method comparison,
the benchmark must know the graphPanel, plotArea, axis endpoints, tick/grid
positions, and numeric label boxes that count as truth.

## Sources Checked

| Source | Relevant point for ChromaLab |
| --- | --- |
| Plot2Spectra GitHub / paper | Treats plot digitization as staged axis alignment, plot-region detection, tick text detection, then plot-line extraction. It also refines plot-region boxes with edge/axis constraints. |
| Plot2Spectra Digital Discovery article | Confirms the practical order: plot region, axes/ticks, then semantic/line extraction. This matches ChromaLab's need to annotate graphPanel/plotArea/axes before trace scoring. |
| Context-Aware Chart Element Detection / CACHED | Chart element detection depends on local and global chart context, not isolated object detection. ChromaLab must annotate rejected non-graph regions and layout context, not only accepted boxes. |
| C3E chart classification/content extraction | Splits chart content extraction into chart classification, text detection/recognition, text-role classification, axis analysis, legend analysis, and data extraction. ChromaLab must separate numeric tick labels from title, legend, ion, and m/z text. |
| ICPR/CHART text-role tasks | Uses explicit text roles such as chart title, axis title, tick labels, legend title, legend label, value label, and other text. ChromaLab's truth labels must preserve similar roles plus chromatogram-specific ion/m/z rejection roles. |

## Required Annotation Fields

| Field | Why it is required |
| --- | --- |
| `expected_physical_graph_count` | Separates one graph, stacked traces, multi-panel pages, and false graph splits. |
| `layout_class` | Carries chromatogram layout semantics into graph-count scoring. |
| `panel_groups` | Records which panels share axes or form separate graph outputs. |
| `graph_panel_bounds` | Full physical graph region, including plot, labels, titles, and context. |
| `plot_area_bounds` | Data-only region used for trace and peak extraction. |
| `axis_endpoints` | Required for axis-line error and scale orientation scoring. |
| `plot_frame_edges` | Required when visible ticks are weak but frame/grid evidence exists. |
| `tick_or_grid_positions` | Required for tick/grid precision, recall, and calibration support. |
| `numeric_label_boxes` | Required for OCR crop scoring and label-to-axis pairing. |
| `text_role_labels` | Prevents title, legend, ion, m/z, and peak-annotation numbers from becoming tick labels. |
| `rejected_non_graph_regions` | Makes graph count failures inspectable instead of hidden by accepted boxes only. |

## Method Comparison Inputs

DR-C method comparison may evaluate:

- current ChromaLab graph/layout stack;
- OpenCV/Rust contour, component, line, Hough, and LSD primitives;
- anchor-free plot-region detectors;
- edge/frame-refined plotArea candidates;
- context-aware chart element detectors;
- text-role assisted axis/tick grouping;
- multi-panel and stacked-trace grouping rules.

DR-C method comparison must not use:

- fixture-specific coordinates;
- expected graph count as a candidate-selection input;
- E2B/VLM pixel geometry;
- E2B/VLM numeric calibration truth;
- release-ready claims from review-only evidence.
